三月 06, 2020
  • 操作系统

作业

作业调度
  1. 记录系统中各作业的状况
  2. 从后备队列中挑选作业投入运行
  3. 为被选中的作业做准备工作，建立进程，分配资源
  4. 作业执行结束时作善后处理工作
调度的实质
  • 资源的分配
调度算法定义
  • 根据系统的资源分配策略所规定的资源分配算法
调度算法评价
  • 调度实质上是一个策略问题，设定的目标往往是相互冲突的。
  • 目标：
    • 单位时间内运行尽可能多的作业
    • 使处理机尽可能保持“忙碌”
    • 是各种I/O设备得以充分利用
    • 对所有的作业都是公平合理的
调度算法的性能准则
  1. 用户的角度
    • （平均）周转时间：作业从提交到完成的时间
    • （平均）带权周转时间：周转时间/CPU运行时间
    • 吞吐量：单位时间内所完成的作业数
    • 处理机利用率：CPU运行时间/总时间
    • 各种设备的均衡利用：如CPU繁忙的作业和I/O繁忙的作业搭配
  2. 处理机角度
    • 响应时间：用户输入一个请求（如击键）到系统给出首次响应（如屏幕显示）的时间
    • 吞吐量：单位时间内所完成的作业数
    • 易于实现
    • 执行开销低：时间、空间复杂度低
  3. 算法本身角度
    • 公平性：不因作业或进程本身的特性而使上述指标过分恶化
    • 优先级：可以使关键人物达到更好的指标
调度算法性能的衡量
  1. 作业周转时间：完成时刻减去进入系统时刻
  2. 平均周转时间：所有作业周转时间相加除以作业数
  3. 带权周转时间：周转时间除以实际运行时间
  4. 平均带权周转时间：所有带权周转时间相加除以作业数
算法

  1. 先来先服务算法

    • 按作业的先后顺序进行调度

    • 处理过程：

      1. 按照作业提交先后次序，分CPU执行
      2. 当前作业占用CPU，指导执行玩或阻塞（如申请I/O）让出CPU
      3. 作业被唤醒后（如I/O完成），不立刻恢复执行，等待当前作业让出CPU后才可以执行

    • 最简单的调度算法

    • 对短作业不利（平均周转时间延长）

  2. 短作业优先

    • 按作业长短进行调度，短作业优先
      • 后来的作业不抢占先正在执行的作业
      • 对预计执行时间短的作业优先分配CPU
    • 目标是减少平均周转时间，是对FCFS算法（先来先服务算法）的改进
    • 相比于FCFS改善平均周转时间和平均带权周转时间
    • 缩短作业的等待时间
    • 提高系统的吞吐量
    • 对长作业不利，可能长时间的不到执行
    • 难以准确估计做的（进程）的执行时间，从而影响调度性能
    • 未能依据作业的紧迫程度来划分执行的优先级

  3. 最短剩余时间优先算法

    • 短作业优先算法的变型，也称作抢占式的短作业优先算法
    • 允许比当前进程剩余时间更短的进程来抢占
    • 抢占时机：新作业加入队列时

  4. 最高响应比有限算法

    • 从就绪队列中选出响应比最高的作业投入执行
    • 响应比R=（等待时间W+要求执行时间T）/要求执行时间T
    • FCFS和SJF的折衷
    • 既照顾了短作业，也考虑到了先后顺序
    • 每次调度时要调用响应比计算，增加了系统开销

  5. 基于优先数调度算法

    • 由用户规定优先数（外部优先数）

      用户提交作业时，根据急迫程度规定适当的优先数

      作业调度程序根据JCB优先数决定进入内存的次序

    • 由系统计算优先数（内部优先数）
