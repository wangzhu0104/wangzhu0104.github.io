操作系统启动过程

  1. BIOS和主引导记录MBR

    • 实模式（实地址模式，real mode）

      • 程序按照8086寻址方法访问0h—FFFFFh（1MB）空间
      • 寻址方式：物理地址（20位）=段地址：偏移地址
      • CPU单任务运行

    • 保护模式（内存保护模式，protect mode）

      • 寻址方式：段（32位）和偏移量（32位），寻址4GB空间

        段的属性：起始地址，存取属性，权限级别。。。

      • 段页式寻址机制（段，页）

      • 虚拟地址，进程，封闭空间

      • 应用程序和操作系统的运行环境都被保护

      • CPU支持多任务

    • 实模式存取的1M空间

      • 前面640k【00000——9FFFF】：基本内存

      • 中间128k【A0000——BFFFF】：显卡显存

      • 末尾256k【C0000——FFFFF】：BIOS

        C0000——C7FFF：显示卡BIOS

        C8000——CBFFF：IDE控制器BIOS

        F0000——FFFFF：最后64kb，系统BIOS（重点）

    • 系统BIOS

      • Basic I/O System(FIrmware,固件)

        • 基本输入/输出系统

        • 位置：F0000-FFFFF

        • 功能：

          1. 系统启动配置

          2. 基本的设备I/O服务

          3. 系统的加电自检和启动

            • 加电自检POST

              1. Power On Self-Test（加电自检）
              2. 初始化基本硬件
                • CPU，内存，显卡
              3. 自检正常不提示，错误则通过喇叭提示

            • 按下PowerOn或者Reset键执行第一条指令

              1. 执行FFFF0处的指令

            • Post之后

              1. 查找显卡BIOS，调用显卡BIOS
              2. 依次查找其他设备执行相应设备的BIOS
              3. 显示启动画面
              4. 从硬盘/软盘/光驱/U盘读入OS
              5. OS启动后，由OS接管计算机

            • 主启动记录MBR

              • MBR
                1. 存放在硬盘/软盘的首扇区
                2. 存放和OS启动的相关信息（Main Boot Record）
                3. 512 BYTES
                4. 结束：0xAA55h

            • MBR/硬盘分区/格式化

            • BIOS和MBR程序运行过程

              1. POST—>CMOS设置（硬盘启动）—>读取MBR—>控制权交给MBR
              2. MBR读取分区表，找到其中的活动分区，并确认其他的分区都不是活动分区。MBR读取活动分区的第一个分区（分区引导记录PBR）并把它加载到内存中去
              3. PBR继续控制后面的引导进程
